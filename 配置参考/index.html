# 基本配置

# 基地

  • 类型： string
  • 默认： / 将部署站点的基本 URL。如果您计划在子路径下部署站点，则需要设置此项，例如 GitHub 页面。如果您计划将站点部署到https://foo.github.io/bar/，base则应将其设置为"/bar/"。它应始终以正斜杠（/）开头和结尾。

它base会自动添加到/以其他选项开头的所有 URL 中，因此您只需指定一次。

也可以看看：

  • 基本URL
  • 部署指南> Github页面

# 标题

  • 类型： string
  • 默认： undefined 该网站的标题。这将是所有页面标题的前缀，并显示在默认主题的导航栏中。

# 说明

  • 类型： string
  • 默认： undefined 该网站的描述。这将在页面 HTML 中呈现为标记。

# 头

  • 类型： Array
  • 默认： [] 要注入页面 HTML 的额外标签<head>。每个标签都可以以。的形式指定[tagName, { attrName: attrValue }, innerHTML?]。例如，要添加自定义图标：
module.exports = {
  head: [['link', { rel: 'icon', href: '/logo.png' }]]
};

# 主机

  • 类型： string
  • 默认： '0.0.0.0' 指定用于开发服务器的主机。

# 端口

  • 类型： number
  • 默认： 8080 指定用于 dev 服务器的端口。

# DEST

  • 类型： string
  • 默认： .vuepress/dist 指定输出目录 vuepress build。

# GA

  • 类型： string
  • 默认： undefined 提供 Google Analytics ID 以启用集成。

小费 请注意 GDPR（2018 年欧盟数据保护规则改革），并考虑将 Google Analytics 设置为在适当和/或需要时匿名化 IP。

＃ serviceWorker 类型： boolean 默认： false 如果设置为 true，VuePress 将自动生成并注册一个缓存内容以供脱机使用的服务工作者（仅在生产中启用）。

如果开发自定义主题，该 Layout.vue 组件也将发出以下事件：

sw-ready sw-cached sw-updated sw-offline sw-error PWA 注意事项

该 serviceWorker 选项仅处理服务工作者。要使您的站点完全符合 PWA 标准，您需要提供 Web App Manifest 和图标.vuepress/public。有关更多详细信息，请参阅有关 Web App Manifest 的 MDN 文档。

此外，只有在您能够使用 SSL 部署站点时才启用此功能，因为服务工作者只能在 HTTPs URL 下注册。

＃语言环境 类型： { [path: string]: Object } 默认： undefined 指定 i18n 支持的语言环境。有关更多详细信息，请参阅国际化指南。

＃ shouldPrefetch 类型： Function 默认： () => true 一个控制哪些文件应该生成资源提示的函数。请参见 shouldPrefetch。

＃主题化 ＃主题 类型： string 默认： undefined 指定此选项以使用自定义主题。使用值"foo"，VuePress 将尝试加载主题组件 node_modules/vuepress-theme-foo/Layout.vue。

＃ themeConfig 类型： Object 默认： {} 为使用的主题提供配置选项。选项将根据您使用的主题而有所不同。

# markdown.lineNumbers

类型： boolean 默认： undefined 是否在每个代码块的左侧显示行号。

另见：

行号

# markdown.slugify

类型： Function 默认值：来源 将标题文本转换为 slugs 的功能。这会影响为标题锚点，目录和侧栏链接生成的 ID /链接。

# markdown.externalLinks

类型： Object 默认： { target: '_blank', rel: 'noopener noreferrer' } 键和值对将添加到指向外部链接的标记。默认选项将在新窗口中打开外部链接。

# markdown.anchor

类型： Object 默认： { permalink: true, permalinkBefore: true, permalinkSymbol: '#' } markdown-it-anchor 的选项。（注意：markdown.slugify 如果要自定义标头 ID，则更喜欢。）

# markdown.toc

类型： Object 默认： { includeLevel: [2, 3] } 对于选择降价，它表-OF-内容。（注意：markdown.slugify 如果要自定义标头 ID，则更喜欢。）

# markdown.config

类型： Function 默认： undefined 用于修改默认配置或将其他插件应用于用于呈现源文件的 markdown-it 实例的函数。例：

module.exports = { markdown: { config: md => { md.set({ breaks: true }) md.use(require('markdown-it-xxx')) } } } ＃构建流水线 配置 CSS 预处理器

＃ postcss 类型： Object 默认： { plugins: [require('autoprefixer')] } postcss-loader 的选项。注意指定此值将覆盖 autoprefixer，您需要自己包含它。

＃手写笔 类型： Object 默认： { preferPathResolver: 'webpack' } 手写笔加载器的选项。

＃ SCSS 类型： Object 默认： {} 用于加载文件的 sass-loader 的选项*.scss。

＃上海社会科学院 类型： Object 默认： { indentedSyntax: true } 用于加载文件的 sass-loader 的选项*.sass。

＃少 类型： Object 默认： {} less-loader 的选项。

＃ configureWebpack 类型： Object | Function 默认： undefined 修改内部 webpack 配置。如果值是 Object，它将使用 webpack-merge 合并到最终配置中; 如果该值是一个函数，它将接收 config 作为第一个参数，并将 isServer 标志作为第二个参数。您可以直接改变配置，也可以返回要合并的对象：
